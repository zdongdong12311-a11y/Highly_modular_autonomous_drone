# OpenCV RGB 视觉识别 + 舵机控制铝合金铁爪

在无人机自主导航基础上，增加**颜色识别**与**舵机爪抓取/投放**功能，实现"识别→抓取→运输→投放"的完整任务闭环。

## 功能概述

| 功能 | 说明 |
|------|------|
| 视觉颜色识别 | 基于 OpenCV HSV 色彩空间，识别红/绿/蓝三色圆形目标 |
| 舵机爪控制 | 通过串口连接 ESP8266 控制铝合金铁爪抓取/释放 |
| 自主导航 | 复用主项目的导航控制器，支持 move_base 避障导航 |
| 航点任务链 | 按 `point.txt` 航点顺序执行，在特定航点触发视觉/爪操作 |

## 工作流程

```
起飞 → 逐航点导航 → 航点4: 识别目标颜色并记忆
                    → 航点5: 抓取小球
                    → 航点11/12/13: 限时识别目标颜色
                        ├─ 识别成功 → 飞往投放点 → 释放爪子 → 降落
                        └─ 识别失败(全部) → 飞回备降航点 → 释放爪子 → 降落
```

## 视觉识别参数

颜色 HSV 阈值（可在 `RobotVision.__init__` 中调整）：

| 颜色 | H 范围 | S 范围 | V 范围 |
|------|--------|--------|--------|
| 红 | 0~10, 170~180 | 150~255 | 70~255 |
| 绿 | 35~85 | 100~255 | 50~255 |
| 蓝 | 100~130 | 130~255 | 50~255 |

### 航点文件格式 (`/home/orangepi/Desktop/test_points.txt`)

```
x y z hover_time
```

| 字段 | 说明 |
|------|------|
| x, y | 目标点坐标 (m) |
| z | 目标高度 (m) |
| hover_time | 到达后悬停时间 (s) |

### 航点特殊逻辑

- **航点 4**: 第一次视觉识别（RGB 全色检测），锁定目标颜色
- **航点 5**: 舵机爪抓取
- **航点 11/12/13**: 第二次视觉识别（仅检测记忆的颜色），识别成功后飞往投放点+释放爪子；全部失败则飞回备降航点
- **备降航点**: 可通过 `FALLBACK_LAND_WP_INDEX` 配置（默认第 11 个航点）

### 二次识别投放偏移

可通过 `DROP_OFFSET_X/Y/Z` 变量在代码中调整投放点相对于当前位置的偏移量。

## 串口爪控制协议

通过串口发送单字节指令：

| 指令 | 动作 |
|------|------|
| `1` | 抓取 (grab) |
| `2` | 张开 (release) |

对应 ESP8266 Arduino 端需实现串口监听，收到 `'1'` 控制舵机转动到抓取角度，收到 `'2'` 转动到张开角度。

## 注意事项

1. 确保串口设备权限：`sudo chmod 666 /dev/ttyUSB0`
2. 首次运行前手持测试爪子动作是否正常
3. 相机建议固定安装，视场角参数 `CAMERA_FOV_H/V` 需根据实际相机填写
4. 建议在地面先使用 `test_points.txt` 测试航点逻辑
